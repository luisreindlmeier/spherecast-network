Review the changes on the current branch for TypeScript errors, bugs, and security issues. Fix anything critical. Then commit all remaining changes and open a PR to main using `gh pr create`.

Fill the PR template (.github/pull_request_template.md) properly:

- What was built: one clear sentence
- Why: the problem it solves
- Changes: bullet list of main changes
- How to test: concrete steps to verify it works
- Screenshots: mention if UI changed

Use a clear conventional commit title: `feat:`, `fix:`, `chore:` etc.
